UserService

UserService manages user registration and profiles.

Base URL: http://localhost:8080

Authentication: All RPCs except Login and CreateUser require a JWT token in the Authorization: Bearer <token> header. Obtain a token via the Login RPC.

Method NameRequest TypeResponse TypeDescription
Login LoginRequest LoginResponse

Login authenticates a user with email and password. Returns a JWT access token on success. Returns UNAUTHENTICATED if credentials are invalid. Public endpoint — no Authorization header required.
CreateUser CreateUserRequest CreateUserResponse

CreateUser registers a new user. Returns ALREADY_EXISTS if the email is taken. Public endpoint — no Authorization header required.
GetUser GetUserRequest GetUserResponse

GetUser fetches a user by ID. Returns NOT_FOUND if the user does not exist or is deleted.
ListUsers ListUsersRequest ListUsersResponse

ListUsers returns a paginated list of active users.
UpdateUser UpdateUserRequest UpdateUserResponse

UpdateUser updates the user's name and email. Returns NOT_FOUND if the user does not exist.
DeleteUser DeleteUserRequest DeleteUserResponse

DeleteUser performs a soft delete on the user. Returns NOT_FOUND if the user does not exist.

ProductService

Authentication: All RPCs require a JWT token in the Authorization: Bearer <token> header.

Method NameRequest TypeResponse TypeDescription
CreateProduct CreateProductRequest CreateProductResponse

CreateProduct registers a new product in the catalog. Returns ALREADY_EXISTS if the SKU is taken.
GetProduct GetProductRequest GetProductResponse

GetProduct fetches a product by ID. Returns NOT_FOUND if the product does not exist or is deleted.
ListProducts ListProductsRequest ListProductsResponse

ListProducts returns a paginated, optionally filtered list of products.
UpdateProduct UpdateProductRequest UpdateProductResponse

UpdateProduct updates product metadata (name, description, price, category). Does NOT change stock. Use UpdateStock for inventory changes.
DeleteProduct DeleteProductRequest DeleteProductResponse

DeleteProduct performs a soft delete.
UpdateStock UpdateStockRequest UpdateStockResponse

UpdateStock atomically adjusts a product's stock level. Uses SELECT ... FOR UPDATE to prevent race conditions. Returns FAILED_PRECONDITION if stock would go negative.
GetStockLevel GetStockLevelRequest GetStockLevelResponse

GetStockLevel returns the current stock and recent movement history.

CreateOrderRequest

CreateOrderRequest places a new order.

The order-service will:

1. Verify the user exists (via user-service)

2. Fetch product details and verify stock (via product-service)

3. Deduct stock for each item (via product-service UpdateStock)

4. Create the order record with status PENDING

OrderStatus

OrderStatus represents the lifecycle of an order.

State transitions:

PENDING -> CONFIRMED (after payment)

CONFIRMED -> SHIPPED -> DELIVERED

PENDING -> CANCELLED (by user or refund)

NameNumberDescription
ORDER_STATUS_UNSPECIFIED 0

ORDER_STATUS_PENDING 1

Order created, awaiting payment.
ORDER_STATUS_CONFIRMED 2

Payment received, order confirmed.
ORDER_STATUS_SHIPPED 3

Order shipped to customer.
ORDER_STATUS_DELIVERED 4

Order delivered to customer.
ORDER_STATUS_CANCELLED 5

Order cancelled. Stock has been restored.

OrderService

OrderService manages customer orders.

Communicates with user-service and product-service via Connect RPC.

Base URL: http://localhost:8082

Authentication: All RPCs require a JWT token in the Authorization: Bearer <token> header. The token is forwarded to upstream services (user-service, product-service) automatically.

Method NameRequest TypeResponse TypeDescription
CreateOrder CreateOrderRequest CreateOrderResponse

CreateOrder places a new order. Calls user-service to verify the user and product-service to check/deduct stock. Returns NOT_FOUND if user or product does not exist. Returns FAILED_PRECONDITION if stock is insufficient.
GetOrder GetOrderRequest GetOrderResponse

GetOrder fetches a single order by ID including line items.
ListOrders ListOrdersRequest ListOrdersResponse

ListOrders returns a paginated list of orders with optional filters.
UpdateOrderStatus UpdateOrderStatusRequest UpdateOrderStatusResponse

UpdateOrderStatus manually transitions order status.
CancelOrder CancelOrderRequest CancelOrderResponse

CancelOrder cancels a pending order and restores stock via product-service. Returns FAILED_PRECONDITION if the order is not in PENDING status.

CreatePaymentRequest

CreatePaymentRequest initiates a payment for an order.

The payment-service will:

1. Fetch order details (via order-service) to get the amount

2. Verify the order belongs to the user

3. Process payment (simulated - always succeeds)

4. Update order status to CONFIRMED (via order-service)

PaymentService

PaymentService processes payments and refunds.

Communicates with order-service via Connect RPC.

Payment processing is simulated (always succeeds) for learning purposes.

Base URL: http://localhost:8083

Authentication: All RPCs require a JWT token in the Authorization: Bearer <token> header. The token is forwarded to order-service automatically.

Method NameRequest TypeResponse TypeDescription
CreatePayment CreatePaymentRequest CreatePaymentResponse

CreatePayment processes a payment for an order. Fetches order amount from order-service and updates order status to CONFIRMED. Returns NOT_FOUND if the order does not exist. Returns PERMISSION_DENIED if the order does not belong to the user.
GetPayment GetPaymentRequest GetPaymentResponse

GetPayment fetches a single payment by ID.
ListPayments ListPaymentsRequest ListPaymentsResponse

ListPayments returns payments filtered by order or user.
RefundPayment RefundPaymentRequest RefundPaymentResponse

RefundPayment issues a full refund and cancels the associated order. Returns FAILED_PRECONDITION if the payment is not in COMPLETED status.
